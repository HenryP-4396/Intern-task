# Python 与 MySQL 综合练习任务报告

## 一、MySQL 数据库与学生表操作

本次练习先安装并配置 MySQL 数据库，复习数据库、表、记录的基本概念，以及增删改查语句。

创建数据库：

```sql
CREATE DATABASE IF NOT EXISTS python_mysql_homework
  DEFAULT CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;
```

创建学生表：

```sql
CREATE TABLE students (
  id INT PRIMARY KEY AUTO_INCREMENT COMMENT '学生编号',
  student_no VARCHAR(20) NOT NULL UNIQUE COMMENT '学号',
  name VARCHAR(50) NOT NULL COMMENT '姓名',
  height DECIMAL(4, 1) NOT NULL COMMENT '身高，单位cm',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
);
```

字段设计：

- `id`：主键，自增，唯一标识每个学生。
- `student_no`：学号，设置唯一约束，避免重复。
- `name`：姓名。
- `height`：身高，使用 `DECIMAL(4,1)` 保存一位小数。
- `created_at`、`updated_at`：记录数据创建和更新时间。

增删改查示例：

```sql
INSERT INTO students (student_no, name, height)
VALUES ('S001', '张三', 170.5);

SELECT * FROM students;

UPDATE students
SET height = 172.0
WHERE student_no = 'S001';

DELETE FROM students
WHERE student_no = 'S001';
```

## 二、Python 与 MySQL 交互

Python 通过第三方库 `pymysql` 连接 MySQL。基本流程是：

1. 建立数据库连接。
2. 创建游标对象。
3. 执行 SQL 语句。
4. 如果是查询，获取查询结果。
5. 如果是增删改，提交事务。
6. 关闭游标和连接。

为了减少重复代码，本次封装了 `MySqlHelper` 类，提供以下方法：

- `query_one(sql, params)`：查询一条数据。
- `query_all(sql, params)`：查询多条数据。
- `execute(sql, params)`：执行一条增删改语句。
- `execute_many(sql, params_list)`：批量执行多条数据。

设计方法参数时，将 SQL 和参数分开传入：

```python
db.execute(
    "UPDATE students SET height = %s WHERE student_no = %s",
    (172.0, "S001"),
)
```

这样做的优点是调用清晰，并且可以避免 SQL 注入问题。

## 三、爬虫基础知识与原理

爬虫的基本原理是模拟浏览器访问网页，然后从网页内容中提取需要的数据。

常见步骤：

1. 使用 `requests` 发送 HTTP 请求。
2. 设置 `User-Agent` 请求头，模拟正常浏览器。
3. 获取服务器返回的 HTML 页面。
4. 使用 `BeautifulSoup` 解析 HTML。
5. 使用 CSS 选择器定位目标元素。
6. 清洗并整理数据。
7. 将数据保存到 MySQL 数据库。

爬虫需要注意：

- 遵守网站规则，不高频请求。
- 添加合理的 `sleep` 延迟。
- 页面结构变化时，需要更新解析规则。
- 有些网站会有反爬策略，可能需要登录、Cookie 或更稳定的数据接口。

## 四、百度热搜 Top10 爬取与入库

百度热搜数据保存到 `baidu_hot_search` 表。

字段设计：

- `rank_no`：热搜排名，用于柱状图或排行榜。
- `keyword`：热搜关键词，用于展示标题或词云。
- `hot_score`：热度值，用于对比不同热搜热度。
- `url`：详情链接。
- `source`：数据来源。
- `crawl_date`：爬取日期，用于后续趋势分析。

可视化思路：

- 使用柱状图展示 Top10 热度。
- 使用折线图展示不同日期同一关键词的热度变化。
- 使用表格展示排名、关键词、热度和详情链接。

## 五、豆瓣电影 Top100 爬取与字段设计

豆瓣电影数据保存到 `douban_movies` 表。

字段设计时考虑了后续可视化展示：

- `rank_no`：排名，可做电影排行榜。
- `title`：电影名称，用于展示。
- `original_title`：原始片名。
- `rating`：评分，可做评分分布图。
- `rating_count`：评价人数，可判断电影热度。
- `quote`：电影短评，可作为卡片文案。
- `directors`：导演。
- `actors`：主演。
- `release_year`：上映年份，可做年代分布统计。
- `country`：国家/地区，可做地区占比。
- `genre`：类型，可做类型分布图。
- `detail_url`：详情页链接。
- `poster_url`：海报链接，适合前端电影卡片展示。
- `crawl_date`：爬取日期。

可视化服务可以展示：

- Top100 评分排行榜。
- 各年份电影数量分布。
- 不同类型电影占比。
- 国家/地区分布。
- 电影海报卡片墙。

## 六、运行说明

安装依赖：

```bash
pip install -r requirements.txt
```

运行学生表示例：

```bash
python src/student_demo.py
```

运行百度热搜爬虫：

```bash
python src/baidu_hot_search_spider.py
```

运行豆瓣电影爬虫：

```bash
python src/douban_top100_spider.py
```

运行前需要确认 `src/config.py` 中的 MySQL 用户名、密码和数据库名与本机一致。
如果数据库不存在，`MySqlHelper` 会先自动创建数据库，再执行后续表操作。
